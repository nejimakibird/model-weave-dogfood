# 0.1.8 - Relationship View and Codeset / State Usage

## Summary

Model Weave 0.1.8 improves model navigation and impact analysis by adding Relationship View enhancements and codeset value usage tracking.

This release focuses on making structured Markdown models easier to inspect, refine, and maintain. It helps users find which models reference each other, which source links are related, and where codeset/state values are used in structured fields.

This release includes the internal 0.1.7 Relationship View milestone and the 0.1.8 Codeset / State Usage improvements.

## Added

### Relationship View / Impact View

* Added Relationship View for inspecting model relationships.
* Shows objects referenced by the current model.
* Shows objects that reference the current model.
* Shows unresolved references.
* Shows related Source Links.
* Added copyable Relationship Summary in Markdown.
* Added Open buttons for related models.
* Added collapsible object rows and source link groups.
* Added Relationship View ON/OFF setting.
* Added lightweight Japanese UI captions for Relationship View.

### Codeset / State Usage

* Added codeset value usage tracking inside Relationship View.
* Codeset references are shown as normal inbound relationships.
* Codeset value references are shown under Value usage when viewing a codeset model.
* Detects qualified codeset value references in structured fields, such as:

  * `[[CODE-ID]].value`
  * `[[path/CODE-ID]].value`
  * `CODE-ID.value`
* Added support for app_process `Flows.condition` value usage detection.
* Added support for screen structured condition fields:

  * `Fields.condition`
  * `Actions.condition`
  * `Messages.condition`
  * Local Process `Steps.condition`
  * Local Process `Errors.condition`
* Integrated value usage into Relationship Summary instead of adding a separate Codeset Usage View.

### V0.8 Format Guidance

* Added V0.8 guidance for structured codeset/state usage.
* Clarified that natural language text is for human explanation.
* Clarified that analyzers use structured table fields only.
* Clarified that Summary, Notes, and arbitrary prose are not parsed for codeset/state usage.
* Clarified Screen and app_process boundaries:

  * app_process can model flows such as business flows, internal process flows, job flows, and batch flows.
  * Screen remains UI/event/action oriented.
  * Screen does not gain app_process Flows in V0.8.
  * Complex flows should be modeled as app_process and referenced from screen Actions.invoke.
* Updated command-generated templates to match V0.8 structured condition guidance.

### Build / Development

* Added GitHub Actions build workflow.
* Added Codex Git operation policy to prevent unintended remote Git operations.

## Changed

* Relationship View rendering was refactored to make the grouped usage UI easier to reuse.
* Source Links are grouped by relation kind, owner model, and path to reduce noise.
* Value usage display is grouped by codeset value instead of being compressed into long inline rows.
* Screen templates now include condition columns for structured UI state/control usage.
* Screen Local Processes template now uses `condition` columns on Steps and Errors instead of a standalone Conditions table.

## Notes

* 0.1.7 was used as an internal development milestone for Relationship View / Impact View.
* 0.1.8 is the next public release candidate and includes both the internal 0.1.7 work and the Codeset / State Usage improvements.
* Codeset value usage detection is intentionally conservative.
* Model Weave does not infer codeset/state usage from arbitrary prose in this release.
* Value usage is detected only from explicit structured references.

## Out of Scope

The following items are not included in this release:

* Natural-language reference extraction from Notes or Summary.
* State transition diagrams.
* Codeset value usage heatmaps.
* Screen Flow support.
* Mermaid renderer expansion.
* Separate Codeset Usage View.
* Automatic parsing of labels or value codes without a codeset identifier.
