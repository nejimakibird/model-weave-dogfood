# Model Weave 0.1.13

Model Weave 0.1.13 focuses on Domain-aware app_process and DFD modeling, release documentation cleanup, and English/Japanese UI consistency.

## Highlights

* Added Domain Sources support for `app_process` and `dfd_diagram`.
* Added `Steps.domain` placement for `app_process` Business Flow, including local `## Domains`, reusable `## Domain Sources`, hierarchical Domain rendering, and Domain group Color Scheme styling.
* Added external Domains plus local Domains merge behavior for DFD `Objects.domain`.
* Aligned Domain Sources table compatibility so supported formats accept both `ref` and `ref | notes`.
* Normalized `domains` / `domain_diagram` `render_mode` handling and settings labels.
* Cleaned up templates and FORMAT docs, including deprecated auto render mode examples and `dfd_object` kind guidance.
* Improved English/Japanese UI consistency in DFD, Class, app_process, Screen, preview summary, impact, and related views.
* Fixed Screen preview clipping when empty sections render placeholder rows.

## Notes

* `Steps.lane` remains supported as legacy-compatible layout-only placement when `Steps.domain` is empty.
* Domain Source references continue to use table order for merge behavior.
* Domain group Color Scheme styling applies to resolved Domain groups; app_process step nodes continue to use `target=app_process` and `Steps.kind`.
