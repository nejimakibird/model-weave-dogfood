# Model Weave 0.1.18

Model Weave 0.1.18 improves the viewer review workflow and Diagnostics experience. It adds stable lower panel tabs, grouped diagnostics actions, bulk Markdown copy, manual-edit guidance, and a small Quick Fix MVP for missing frontmatter `id` / `name`. Fully empty Markdown table data rows are now ignored as editing noise, while partially filled rows are still validated.

## Compatibility

Model Weave 0.1.18 requires Obsidian 1.8.7 or later.

## Highlights

* Added stable lower panel tabs for review information.
* Applied lower panel tabs to both graph and non-graph model review views.
* Kept lower tabs stable across renderer switching and Business Flow direction changes.
* Improved Diagnostics tab layout and grouped actions.
* Added bulk Markdown copy actions for all diagnostics, errors, warnings, and notes.
* Added manual-edit guidance to diagnostic details.
* Added a Quick Fix MVP for missing frontmatter `id` and `name`.
* Derives missing `id` from the file basename when safe.
* Preserves existing frontmatter values and does not overwrite them.
* Does not auto-fix missing `type`.
* Fully empty Markdown table data rows are now ignored as editing noise.
* Partially filled rows still produce required-field diagnostics.

## Diagnostics and Quick Fix notes

Quick Fix actions remain intentionally narrow. The MVP only inserts missing frontmatter `id` / `name` when a safe value can be derived, and it re-reads the file before applying the change. Markdown files remain the source of truth.

Bulk diagnostics copy is designed for review and AI/chat workflows. It can copy all diagnostics or only errors, warnings, or notes as Markdown.

## Table editing notes

Fully empty Markdown table data rows are ignored. A row is considered fully empty when every cell is empty or whitespace after trimming. Such rows do not create model rows and do not trigger required-field diagnostics.

Partially filled rows are still validated. For example, a row with an empty `id` or `ref` but non-empty `notes` or other cells remains invalid when that field is required.
